## 1. 竞品深度分析

### 1.1 Arcaea（韵律源点）

**开发商：** lowiro（英国） | **平台：** iOS / Android / Nintendo Switch | **首发：** 2017年

#### 核心玩法机制

Arcaea 采用**双轨道立体空间设计**，是移动音游中最具标志性的创新之一：

- **地面轨道（Ground Lane）**：屏幕下方，传统纵向下落式音符
- **天空轨道（Sky Lane）**：屏幕上方，需以不同触点操作

| 音符类型 | 操作方式 | 说明 |
|---|---|---|
| Tap Note | 单点触击 | 最基础音符 |
| Hold Note | 长按 | 按住不放直到音符结束 |
| Arc Note | 跟随滑动 | 彩虹弧线，手指沿弧线滑动（最标志性音符） |
| Sky Note | 点击 Arc 上的节点 | Arc 弧线上的独立音符 |
| Trace（虚弧） | 跟随但无需精确时机 | 引导型弧线，不计入判定 |

Arc Note 要求手指在三维空间中移动，形成立体操控感，这是 Arcaea 区别于所有竞品的核心差异。

**特殊系统：**
- **搭档系统（Partner System）**：选择不同搭档改变 gauge 类型、技能加成，甚至影响判定音效
- **World Mode**：地图探索系统，通过打歌积累能量推进地图，解锁新曲与章节
- **Link Play**：最多 4 人联机实时对比分数

#### 判定系统设计

| 判定 | 时间窗口 | 说明 |
|---|---|---|
| **Shiny Pure** | ±25ms | 最高判定，额外 +1 分 |
| **Pure** | ±50ms | 普通 Pure，无额外加分 |
| **FAR** | ±100ms | 次级判定，得一半分值 |
| **LOST** | >100ms | Miss，0 分，断 Combo |

**分数计算：**
```
满分 = 10,000,000 分（固定上限）
每个音符基础分 = 10,000,000 / N（N为总音符数）
PM 分数 = 10,000,000 + 总音符数（全 Shiny Pure）
```

**评级：** PM（Pure Memory）> FR（Full Recall）> EX+ (9,900,000) > EX (9,800,000) > AA > A > B ~ D

**Potential 系统**（独创玩家实力指标）：
```
Potential = (最佳30首 + 最近30首中最佳10首) / 40
```

#### 谱面编辑逻辑

- 格式：**.aff（Arcaea File Format）**，纯文本，记录时间戳、音符类型、Arc 贝塞尔曲线控制点坐标
- 难度分级：EZ / HD / FTR / ETR（Eternal，最高难度 12+）
- **官方不支持 UGC**，社区通过 ArcCreate 等工具自制谱面

#### 美术风格与 UI 设计

- **配色基调**：深紫色、午夜蓝、琥珀橙，忧郁梦幻的二次元少女风格
- **角色设计**：主角光（Hikari，白发蓝眼）与对立（Tairitsu，黑发红眼）形成鲜明对比
- **UI 设计**：旋转星球/封套唱片的立体选曲界面，极简信息密度，强调沉浸感
- 每首歌曲配有独立高质量曲绘，画风横跨萌系、写实、暗黑

#### 音乐版权策略

**独立音乐人聚合模式**为主：
- 原创委托（主体）：向知名独立音乐人委托专属曲目
- 授权引进：与 HARDCORE TANO*C、Silentroom 等厂牌合作
- 联动：Groove Coaster、CHUNITHM、maimai、SDVX、Lanota、DEEMO 等

代表艺术家：Camellia、xi、t+pazolite、Eliminate 等。总曲库超过 **350 首**。

#### 商业模式

**基础免费 + 单曲/曲包买断：**

| 付费项目 | 价格区间 |
|---|---|
| 基础免费内容 | 约 30 首免费曲目 |
| 单曲购买（Memory Archive） | 约 6-12 元/首 |
| 曲包（Song Pack） | 约 18-30 元/包 |
| 搭档购买 | 约 18-30 元 |

**无订阅制、无体力系统、无抽卡**，所有付费内容均为永久解锁。

#### 用户口碑与差评痛点

**评分：** App Store 4.8/5 | TapTap ~9.0

**好评核心：** 收录曲豪华、Arc 音符系统创新性极高、美术精良、UI 流畅、付费模式透明

**差评痛点：**
- 曲包价格偏高，全解锁成本数百元
- 部分联动曲目限时，错过后无法购买
- Sky Note 操作门槛极高，新手曲线陡峭
- 部分设备音频不同步
- World Mode 体验低效
- 剧情叙事过于隐晦

### 1.2 Cytus / Cytus II

**开发商：** Rayark Inc.（雷亚游戏，台湾） | **平台：** iOS / Android | **Cytus II 首发：** 2018年

#### 核心玩法机制

**标志性机制——扫描线（Active Scan Line）：**

屏幕中央有一条扫描线，持续在上下方之间来回移动，速度与 BPM 同步。玩家在扫描线经过音符时操作。

Cytus II 在初代基础上大幅扩展：
- 扫描线速度可在歌曲中途实时变化（变快变红，变慢变绿）
- **无固定轨道**，音符在屏幕任意位置出现

| 音符类型 | 操作方式 |
|---|---|
| Click Note | 扫描线经过时单点 |
| Hold Note | 长按至扫描线离开 |
| Drag Note | 沿连续轨迹滑动 |
| Flick Note | 快速滑动（四向） |
| Long Hold | 超长 Hold，跨越扫描线多个来回 |
| Slide Note | 曲线滑动（高难谱面特有） |

#### 判定系统设计

**双重评分体系：**

| 判定 | 分数比例 | TP 贡献 |
|---|---|---|
| C.Perfect（彩虹） | 100% | 100% |
| Perfect（白色） | 100% | 70% |
| Good | 30% | 30% |
| Bad / Miss | 0% | 0% |

Good 在 Cytus II 中被大幅削弱（仅 30% 分值），有意拉大技术区分度。

- **综合分**：满分 1,000,000，及格线 700,000
- **TP 系统**：纯时机准确度，不受 Combo 影响，满 TP 100% = 全 C.Perfect
- **难度上限**：16 级，难度层级 Easy/Hard/Chaos/Glitch/Crash/Dream

#### 谱面编辑逻辑

- 格式：.c2 专有格式，含扫描线速度变化事件、音符坐标和轨迹
- **不支持官方 UGC**，社区通过 **Cytoid**（第三方平台）实现 UGC 生态

#### 美术风格与 UI 设计

- **赛博朋克 + 未来都市 + 二次元**美学体系
- 游戏内模拟真实 OS 界面（iM 虚构社交平台），每个角色有独立桌面/文件/消息
- 配色深邃蓝黑、霓虹色为主，科技感强
- 角色由知名插画师设计：PAFF（偶像 AI）、Nora（工业风机器人）、Ivy（赛博朋克）

#### 音乐版权策略

**角色为核心的音乐包模式**：每位角色对应现实中的特定音乐人/厂牌，游戏内容与真实音乐人 IP 深度绑定。总曲库超过 **300 首**。

#### 商业模式

**角色包买断制：**

| 项目 | 价格参考 |
|---|---|
| 游戏本体 | 免费下载 |
| 角色章节包（20-30 首 + 剧情） | 约 30-68 元/包 |
| 捆绑优惠包 | 约 128-198 元/套 |
| CAPSO! 扭蛋 | 少量内购（有争议） |

无体力系统、无广告、无强制抽卡。

#### 用户口碑与差评痛点

**评分：** App Store ~4.7/5 | TapTap ~8.8-9.0

**好评：** 扫描线系统沉浸感强、iM 叙事方式创意独特、音乐质量高、TP 系统对高端玩家有持续追求动力

**差评：**
- 全解锁成本数百至千元
- CAPSO! 扭蛋争议（随机付费性质）
- 2023 年 AI 美术争议（裁撤美术团队）
- 部分角色剧情完成度不均
- 国服版本更新滞后

### 1.3 Phigros

**开发商：** Pigeon Games（鸽游网络，中国） | **平台：** iOS / Android | **首发：** 2019年

#### 核心玩法机制

**核心创新——动态判定线（Dynamic Judgment Line）：**

判定线可以**旋转（任意角度）、平移、缩放、弯曲、分裂成多条、跳跃**，运动与音乐节奏完美同步，形成"判定线在跳舞"的视觉奇观。被定位为**"无轨道音游"**。

| 音符类型 | 操作 |
|---|---|
| Tap Note | 单点 |
| Hold Note | 长按 |
| Flick Note | 快速滑动（任意方向） |
| Drag Note | 沿轨迹滑动 |

**特殊机制：**
- 高难谱面可同时存在**多条判定线**
- 部分 SP 谱面要求玩家**物理旋转设备**
- 致敬谱面（如 Dead Soul）模拟 VOEZ、MuseDash、DJMAX、jubeat 等不同音游机制

#### 判定系统设计

| 判定 | 分数 | ACC |
|---|---|---|
| Perfect | 100% | 100% |
| Good | 65% | 65% |
| Bad / Miss | 0% | 0% |

判定窗口未官方公布，社区估算 Perfect 约 ±80ms（相对宽松）。

**分数 = ACC 分（900,000 上限）+ Combo 奖励分（100,000 上限）**

评级：**Phi（全 Perfect）** > V（FC）> S (920,000) > A > B > C > F

#### 谱面编辑逻辑

- 格式：phiChart（JSON 结构），记录判定线运动关键帧 + 音符列表
- **不支持官方 UGC**，社区通过 **Phira** 平台分发自制谱

#### 美术风格与 UI 设计

- **清新系二次元风格**，以白色、浅灰、淡蓝、橙黄为主
- 吉祥物为小鸽子，人物角色可爱圆润
- UI 简洁，以"音符"和"星图"为视觉主题
- 整体朴实清晰，不如 Arcaea/Cytus II 设计个性强烈

#### 音乐版权策略

- 大量收录 **BMS 社区**电子音乐（如 Camellia、xi 等）
- 委托国内外独立音乐人原创
- 总曲库超过 **200 首**，持续免费更新

#### 商业模式

**完全免费，无任何付费点——业界最特殊的案例之一。**

| 项目 | 内容 |
|---|---|
| 游戏本体 | 完全免费 |
| 全部曲目 | 完全免费 |
| 内购 / 广告 / 订阅 | **不存在** |

通过章节解锁进度（需达一定评级推进）形成自然驱动力。

#### 用户口碑与差评痛点

**评分：** App Store ~4.9/5 | TapTap ~9.4-9.6（音游品类最高之一）

**好评：** 完全免费是最大好评点、动态判定线创意惊人、谱面艺术性极强、体验纯粹无商业打扰

**差评：**
- 更新速度慢（"鸽游"双关）
- BUG 修复迟缓
- 剧情推进缓慢
- **无竞技/社交系统**
- 多判定线谱面视觉过于混乱

### 1.4 MuseDash（喵斯快跑）

**开发商：** PeroPeroGames（中国） | **发行：** X.D. Network | **首发：** 2018年

#### 核心玩法机制

2D 横版跑酷节奏游戏，操作极简：
- 仅有 **2 个操作区域**（屏幕上下两半），是同类游戏中最简约的设计
- 音符从右侧飞来，点击/长按消灭敌人
- 存在 Boss 障碍物，打错或未打扣血

**特殊机制：**
- **Fever 系统**：积累后激活，角色强化
- **角色 + 精灵双卡系统（Muse + Elfin）**：不同组合提供不同 Buff，直接影响最终分数
- **隐藏难度**：Hard 谱 S 级后解锁 Master

#### 判定系统设计

| 判定 | 窗口 | 得分比例 |
|---|---|---|
| Perfect | ±50ms | 100% |
| Great | ±130ms | 50% |
| Miss | >130ms | 0%，扣血 |

角色 + 精灵搭配 Buff 可叠加分数倍率，**排行榜分数受配装影响极大**。

#### 谱面编辑逻辑

- 私有格式，未公开规范
- **不支持官方 UGC**，社区通过 **MDMC（mdmc.moe）** 平台分享自制谱

#### 美术风格与 UI 设计

- 2D 日系二次元扁平卡通风格，色彩明亮饱和，Cute/Chibi 路线
- 主要角色：凛（Rin）、玛莉嘉（Marija）、布若（Buro），均为萌系少女造型
- 击中特效浮夸鲜艳，视觉反馈直观

#### 音乐版权策略

原创 + 授权 + 联动混合策略：
- 大量 BMS 社区曲目
- 与 Blue Archive、初音未来等大 IP 联动付费曲包
- 涵盖 EDM、Electro Pop、Future Bass、Hard Rock 等

#### 商业模式

| 阶段 | 模式 |
|---|---|
| 2018-2023 | "计划通"买断（约 30 元，含所有未来内容） |
| 2023.10 起 | Muse Plus 订阅制（~$4.99/月），DLC 曲包另付 |

**"计划通"事件是 2023 年最大舆论风波**——官方违背"一劳永逸"承诺，评分暴跌。

#### 用户口碑与差评痛点

**评分：** App Store ~4.6/5 | TapTap 从 9.5+ 暴跌后部分回升

**好评：** 操作简单上手快、音乐品味好、画风可爱

**差评：**
- "计划通"违诺事件（核心信任危机）
- DLC 叠加付费，总成本上升
- 难度上限偏低（Max 11），硬核玩家刺激感不足
- 排行榜"配装碾压技术"

### 1.5 Project Sekai（世界计划）

**开发商：** Colorful Palette / SEGA | **首发：** 2020年（日本），2025年3月中国大陆简中服

#### 核心玩法机制

标准下落式音游，借鉴 SEGA 街机音游 CHUNITHM 设计语言：
- 点击、长按、滑动、Flick（弹划）四种操作，支持多指并发

| Note 类型 | 操作 |
|---|---|
| Tap / Critical Tap | 点击（Critical 为黄色高分 Note） |
| Flick / Critical Flick | 弹划 |
| Hold | 长按 |
| Slide / Critical Slide | 跟随路径滑动 |
| Trace | 跟手滑过即可 |

**特殊机制：**
- **Virtual Live**：实时在线多人演唱会
- **Multi Live**：多人联机合奏
- **Card Party**：5 张卡牌组队，属性技能直接影响得分

#### 判定系统设计

| 判定 | 得分权重 |
|---|---|
| Perfect | 100% |
| Great | 70% |
| Good | 50% |
| Bad / Miss | 0% |

**总分受卡牌 Power 值和技能效果直接影响**——氪金强度直接影响排行榜排名。

难度分级：Easy / Normal / Hard / Expert / Master / **Append**（最高）

#### 谱面编辑逻辑

- SEGA 私有格式，社区格式为 **.sus**（Sliding Universal Score）
- 社区工具 **MikuMikuWorld** / **MMW4CC** + **Sonolus** 平台形成活跃 UGC 生态

#### 美术风格与 UI 设计

- 日系现代动漫画风，5 个组合各有主题色
- 20 名人类角色 + 6 位 Virtual Singer（初音未来等）
- 每次活动更新限定卡面立绘，卡面艺术质量极高

#### 音乐版权策略

| 来源 | 说明 |
|---|---|
| 经典 Vocaloid 曲目 | 授权知名 P 主作品 |
| 游戏原创委托曲 | 专为各组合故事创作 |
| VOCALOID Collection 联动 | 年度词曲大赛优胜曲 |

可切换"原版 Vocaloid 版"和"组合 SEKAI 版"。曲库超过 **250 首**。

#### 商业模式

**免费游玩 + Gacha 抽卡：**
- 限定卡池概率约 3%
- 全球累计收入超过 **8 亿美元**，SEGA 移动端营收支柱

#### 用户口碑与差评痛点

**评分：** App Store ~4.5-4.7/5 | 日本 iOS 畅销榜常驻前 20

**好评：** Vocaloid 曲库丰富、卡面和剧情质量高、谱面有挑战性、Virtual Live 社交感强

**差评：**
- **Gacha 被批"掠夺性"**，限定卡无保底
- **Pay-to-Win 排行榜**
- 双周一次活动强度导致疲劳
- 部分热门 P 主作品因版权无法纳入

### 1.6 Lanota

**开发商：** Noxy Games（台湾） | **首发：** 2016年

#### 核心玩法机制

**核心创新——"转盘"（Notalium Tuner）设计：**

- 中央大型圆形转盘，Note 从圆盘边缘飞入
- **转盘动态变化**：旋转、缩放、平移，玩家必须随时适应
- Note 可从 360 度任意方向出现，无传统轨道概念

| 操作 | 说明 |
|---|---|
| Tap | 到达边缘时点击 |
| Catch | 跟随 Note 在边缘滑过 |
| Flick | 向特定方向弹划 |
| Hold | 按住跟随持续时间 |
| Rail | 长条持续音符 |

#### 判定系统设计

| 判定 | 得分比例 | Combo |
|---|---|---|
| Harmony | 100% | 维持 |
| Tune | ~43% | 维持 |
| Fail | 0% | 断 |

连续 15 个 Fail 导致曲目失败。满分 1,000,000。难度 1-16+。

#### 谱面编辑逻辑

- 私有格式，未公开。不支持 UGC。
- 超过 **669 首歌曲**（含订阅章节）

#### 美术风格与 UI 设计

- **绘本（Picture Book）风格**，手绘水彩质感，温暖柔和
- 主角 Fisica 和 Ritmo，童话风造型
- 故事以翻页绘本形式呈现，音乐与叙事深度绑定
- 获 2016 IMGA SEA "音频卓越奖"第一名

#### 音乐版权策略

### 1.7 Rotaeno（旋转音律）

**开发商：** 独立团队 / X.D. Network 发行 | **首发：** 2022年

#### 核心玩法机制

**核心创新——手机物理旋转作为输入：**

- 传统判定线改为**圆圈（Judgment Circle）**
- Note 从圆圈中心向外飞出

### 2.4 差异化切入点建议

#### 玩法创新方向

**方向 A：「动态谱面」—— 视觉层实时响应音频**
谱面判定逻辑不变，但背景、轨道颜色、粒子效果根据音频频谱实时动态生成。每次游玩视觉层不同，沉浸感大幅提升且不影响公平性。

**方向 B：「情绪感知难度」—— AI 实时调节**
根据玩家连击状态、失误模式，AI 实时微调 Note 密度，让玩家始终处于心流区间。类似节奏地牢的 adaptive music 概念。

**方向 C：「协作谱面」—— 多轨合奏**
两名玩家分别承担不同乐器轨道（一人打鼓轨、一人打旋律轨），合奏完成曲目，失误影响对方音效。

#### 美术风格差异化

**机会窗口**：国风水墨 x 现代赛博朋克融合在音游领域几乎空白。参考《原神》将国风+奇幻完美融合的国际接受度，音游可探索水墨粒子效果、书法判定线、传统乐器采样等方向。但考虑到本项目定位为"日系中二风"，可以将中二美学与东方神秘主义结合。

#### 社交机制创新

**谱面工作坊生态（核心差异化）**：
```
创作者 → 上传自制谱 → 社区审核投票 → 精选库
              ↓
       创作者获得游戏内收益
              ↓
       创作者等级体系（类似 Steam Workshop）
```

**音乐房间社交**：
- 创建/加入实时音乐房间，房主选曲，多人同时游玩
- 实时看到彼此连击数，支持语音连麦

#### UGC 生态建设路径

```
Phase 1（MVP）：内置可视化谱面编辑器 + 本地音乐导入 + 本地分享
Phase 2（社区化）：谱面云端上传/搜索 + 点赞/收藏/评论
Phase 3（生态激励）：质量评分机制 + 创作者认证 + 游戏内货币奖励
```

#### AI 赋能机会

| AI 功能 | 技术可行性 | 玩家需求 | 优先级 |
|---|---|---|---|
| **AI 自动生成谱面** | 高 | 极强 | P0 |
| AI 辅助谱面优化 | 高 | 强 | P1 |
| 智能难度推荐 | 高 | 强 | P1 |
| AI 生成背景音乐 | 中 | 中 | P2 |
| AI 实时调节难度 | 中高 | 中 | P2 |

**AI 自动生成谱面是最强差异化武器**：导入任意一首歌，秒生成可游玩谱面——当前市场完全空白。

### 2.5 执行建议

**定位锚点**：RhythmAether 应定位为**「创作者优先的开放音游平台」**，而非又一款"更好看的下落式音游"。

核心差异化三件套：
1. **AI 谱面生成**：导入任意音乐 → 秒生成可游玩谱面（行业首创）
2. **内置谱面编辑器**：手机端完整创作体验（填补市场空白）
3. **社区谱面工坊**：Steam Workshop 式的创作者生态（建立护城河）

#### 各引擎深度分析

**Unity**：AI 代码生成质量最高，AudioMixer + FMOD 是音频黄金标准（延迟可控 10ms 内），但编辑器庞大不利于快速迭代，Runtime Fee 有不确定性。

**Godot 4.x**：完全开源 MIT，GDScript 简洁 AI 生成准确率高，场景树结构天然适合音游，F5 热运行迭代极快。4.3+ 音频同步已大幅改善。

**Phaser 3**：纯 Web 技术栈，零构建部署，Web Audio API 可实现样本级时间同步，Vite + TS 开发体验极佳。但移动端需 Capacitor 封装。

**PixiJS / Three.js**：前者是纯渲染引擎需大量自建基础设施；后者对 2D 音游过度复杂。均不推荐。

### 3.2 音频处理方案

#### 低延迟音频同步

**Web Audio API 方案（Phaser）：**
```javascript
const audioCtx = new AudioContext({ latencyHint: 'interactive', sampleRate: 44100 });
const startTime = audioCtx.currentTime + 0.1;
source.start(startTime);

// 精确位置计算（精度可达 0.1ms）
function getSongPosition() {
  return (audioCtx.currentTime - startTime) * 1000;
}
```

**Godot 4.x 方案：**
```gdscript
# 精确播放位置补偿公式
func get_accurate_position() -> float:
    return audio_player.get_playback_position() \
         + AudioServer.get_time_since_last_mix() \
         - AudioServer.get_output_latency()
```

**Unity DSP 方案：**
```csharp
double dspStartTime;
void StartMusic() {
    dspStartTime = AudioSettings.dspTime + 0.1;
    audioSource.PlayScheduled(dspStartTime);
}
float GetSongPosition() {
    return (float)((AudioSettings.dspTime - dspStartTime) * 1000f);
}
```

#### 节拍检测库

| 库 | 语言 | 精度 | 推荐场景 |
|---|---|---|---|
| **librosa** | Python | 高 | 服务端首选 |
| **madmom** | Python | 极高（深度学习） | 服务端精化 |
| **Essentia** | Python/C++ | 极高 | 工业级分析 |
| **BeatNet** | Python | 极高 | 实时检测 |
| **meyda** | JavaScript | 中 | Web 端推荐 |

```
  → librosa 提取 BPM + 节拍位置
  → madmom 精化 onset 检测

```

### 3.3 谱面格式设计

分析现有格式（osu! .osu、BMS、Arcaea .aff、Phigros phiChart）后，推荐设计 **RACF（RhythmAether Chart Format）**：

```json
{
  "$schema": "https://rhythmaether.dev/chart-schema/v1",
  "version": "1.0.0",
  "metadata": {
    "title": "歌曲名称",
    "artist": "艺术家",
    "charter": "谱面作者",
    "difficulty": { "label": "Ethereal", "level": 12 },
    "audioFile": "audio.ogg",
    "coverFile": "cover.jpg",
    "previewStart": 30.5,
    "previewEnd": 45.0
  },
  "timing": {
    "offset": 150,
    "bpmChanges": [
      { "beat": 0, "bpm": 174.0, "timeSignature": [4, 4] }
    ]
  },
  "lanes": { "count": 4, "style": "arc" },
  "notes": [
    { "id": "n001", "type": "tap", "beat": 0.0, "lane": 1, "x": 0.25 },
    { "id": "n002", "type": "hold", "beat": 1.0, "endBeat": 3.0, "lane": 2 },
    {
      "id": "n003", "type": "arc",
      "beat": 4.0, "endBeat": 8.0,
      "startX": 0.0, "endX": 1.0,
      "curveType": "sine",
      "arcNotes": [{ "beat": 5.0 }, { "beat": 6.0 }]
    },
    { "id": "n004", "type": "flick", "beat": 8.0, "lane": 3, "direction": "up" }
  ],
  "events": [
    { "type": "speedChange", "beat": 0, "speed": 1.0 },
    { "type": "cameraShake", "beat": 32, "intensity": 0.3, "duration": 0.5 }
  ],
  "generationMeta": {
    "isAIGenerated": false,
    "aiModel": null,
    "humanEdited": true
  }
}
```

**设计原则：**
1. Beat 为基本时间单位（非毫秒），BPM 变化时无需重算所有 Note
2. 完全 JSON，AI 代码生成友好
3. 坐标系归一化（0.0-1.0），适配任意分辨率
4. Note 类型通过 `type` 字段可扩展
5. Events 系统与音符解耦
6. 内置 `generationMeta` 标记 AI 生成来源

### 3.4 推荐技术栈组合

## 5. 风格与世界观方向

### 5.1 三套世界观概念设定

#### 概念 A：「以太共鸣」（Aether Resonance）

**核心设定：**
在一个音律即是物理法则的平行世界中，"以太"（Aether）是连接所有生命的无形能量。这个世界曾经由七位"律者"（Harmonizers）共同守护，每位律者掌管一种音律维度。某日，"沉默灾厄"（The Silence）降临，将世界的音律逐渐吞噬，律者们纷纷陨落或失踪。玩家作为"觉醒者"（Awakener），拥有罕见的"以太共鸣"能力——通过演奏/战斗来重新唤醒沉睡的音律。每一次演奏都是与"沉默"的对抗，每一个 Perfect 判定都在修复世界的裂痕。

**视觉关键词：**
- 水晶碎片与音波纹理
- 发光的符文阵/五线谱融合魔法阵
- 深空紫 + 以太蓝 + 金色高光
- 角色服装融合乐器元素与战斗铠甲

**情感基调：**
史诗宿命感 + 孤独中的希望。类似《Fate》系列的中二燃向情感——"我以音律为剑，斩断沉默"。

---

#### 概念 B：「逆位乐章」（Inverse Opus）

**核心设定：**
现实世界中存在一个与之镜像对称的"逆位世界"（Inverse），在那里，情感以可视化的音乐形态存在。每个人在逆位世界都有一个"音影"（Sound Shadow），是其内心深处情感的具象化。主角是一名能同时感知两个世界的少年/少女——一只眼睛看到现实，另一只看到逆位。"失调者"（Discordants）是情感崩坏的音影，化为危险的存在。主角通过"演奏"来净化失调者，实际是帮助现实中对应的人化解内心创伤。

**视觉关键词：**
- 黑白分界线 / 镜像对称构图
- 墨水在水中扩散的流动感
- 左半边单色（现实），右半边多彩（逆位）
- 角色设计：半边日常服饰 + 半边中二战斗装

**情感基调：**
内省与治愈的中二。类似《Angel Beats!》或《四月是你的谎言》——音乐作为情感创伤的出口，每首歌背后都是一个角色的内心故事。

---

#### 概念 C：「七弦天命」（Septchord Destiny）

**核心设定：**
远未来的漂浮都市群，人类通过植入"音核"（Sonic Core）获得超凡能力。七座城市各以七个基本音阶命名（Do 城、Re 城...Si 城），每座城市的音核技术走向不同方向。"不协和事件"（Dissonance Incident）使七城之间的音律频率开始互相干扰，引发城市间冲突。主角所在的"#（升号）"地下组织试图通过找到"原初和弦"来重新校准世界的频率。每首歌曲对应不同城市的文化风格（从电子到古典），每次游玩即是一次"频率校准任务"。

**视觉关键词：**
- 霓虹朋克 × 音乐科技融合
- 全息投影的音符 UI
- 每座城市独立色系（Do=红、Re=橙...Si=紫）
- 角色设计：赛博义体 + 乐器武装
- 大量频率波形/声谱图作为装饰元素

**情感基调：**
热血反叛 + 科幻冒险。类似《Guilty Crown》或《超时空要塞 Macross》——用音乐改变世界的中二浪漫，每首歌是一场城市革命。

---

| **叙事深度潜力** | ★★★☆☆ | ★★★★★ | ★★★★☆ |
| **美术差异化** | ★★★☆☆ | ★★★★★ | ★★★★☆ |
| **与音游玩法结合度** | ★★★★★ | ★★★★☆ | ★★★★★ |

**推荐方案 B「逆位乐章」**作为首选——最强的美术差异化（黑白分界镜像视觉在音游中完全未见）、最深的叙事潜力（每首歌 = 一个角色故事）、与"中二"定位契合但不过度堆砌设定。

### 5.4 UI/UX 设计方向建议

#### 核心原则

1. **"演奏感"优先于"游戏感"**：UI 应该让玩家感觉自己在演奏音乐，而非玩手机游戏。减少按钮数量，增加手势交互。

2. **零秒等待**：参考 Arcaea 的"15 秒内开始打歌"设计。从启动到第一首歌开始演奏，不超过 3 次点击。

3. **沉浸式选曲**：选曲界面不是"列表"而是"空间"。音乐在切换歌曲时无缝过渡预览，视觉背景随歌曲主题变化。

#### 具体建议

**游玩界面：**
- 判定线设计需有辨识度（建议发光 + 微弱粒子拖尾）
- Note 使用高对比色，确保在任何背景下可读
- 背景模糊化处理（不干扰 Note 辨识），但保留动态效果
- 判定反馈：Perfect = 金色波纹 + 屏幕微震 + 音效；Good = 白色波纹；Miss = 暗红碎裂

**选曲界面：**
- 封面卡片式布局，支持横向滑动
- 当前选中歌曲自动播放预览片段
- 难度选择使用颜色区分（绿 → 蓝 → 紫 → 红 → 金）

**结算界面：**
- 大数字分数 + 评级图标居中
- 各判定数量以环形图展示
- 新纪录时有特殊动画庆祝
- "重试"按钮置于最显眼位置（音游玩家高频操作）

**通用 UI：**
- 字体选择：无衬线中文字体（如思源黑体）+ 定制化数字字体
- 动效：所有界面切换使用流畅过渡（淡入淡出 + 平移），避免生硬跳转
- 适配：优先竖屏设计，兼容横屏
